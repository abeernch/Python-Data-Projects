# Tutorial

This repository is a portfolio container. The workflow is simple:

1. Create a folder for one project.
2. Add a project-specific `README.md`.
3. Store notebooks in `notebooks/` and reusable code in `scripts/`.
4. Put any figures, charts, or screenshots in `docs/images/` or a project-local
   `figures/` folder.
5. Write down the exact Python version and packages required.
6. Include one example command or notebook cell that reproduces the main result.

## Recommended project README sections

- Problem statement
- Data sources
- Setup
- Analysis steps
- Results
- Limitations
- Reproducibility notes

## Practical rule

If a new folder does not have an explanation, the repository will drift back
into a collection of unnamed uploads. Keep the documentation next to the code.
