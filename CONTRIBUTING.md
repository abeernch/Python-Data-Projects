# Contributing

Contributions are welcome when they improve a project`s correctness,
reproducibility, documentation, or presentation.

## Expectations

- Keep changes scoped to one project or one documentation task.
- Replace placeholder language with concrete project details.
- Include installation and run steps for any code that is not obvious.
- Add figures or outputs when they help explain the result.
- Keep filenames and headings descriptive.

## Before opening a pull request

1. Make sure the README matches the actual files in the project folder.
2. Remove any fake example names, placeholder URLs, or template text.
3. Check that any screenshots or figures are readable at their displayed size.
4. Verify that the documented commands still work from a clean checkout.
